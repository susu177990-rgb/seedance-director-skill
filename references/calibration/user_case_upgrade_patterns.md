# User Case Upgrade Patterns

This file extracts direct upgrade patterns from你提供的优秀提示词案例（抒情MV真人电影感系列）用于高质量重写。目标是把“完整的模板填充”升级为“导演级细节密度”。

## Pattern 1: 先搭建项目级系统，再切分镜

- 先写固定世界观：时间、地点、天气/环境、材质温度、主空间层级。
- 写出角色起始形象（发型/服装/姿态）和关系状态。
- 写出道具生命周期（谁拿、放在什么位置、何时变化）。
- 设定禁止项（不出现的关系、动作、镜头逻辑）。

Use case:
- 这个模式能避免“只有一个动作反复拉长”的问题。

## Pattern 2: 分镜采用微事件链而非单句描述

每个镜头都包含：
- 起始状态
- 触发动作
- 反应（含速度/停顿）
- 关系或材质变化
- 落点（镜头可停住的位置）

Use case:
- 复杂动作、情绪变化、产品动作都要有“前-后-结果”。

## Pattern 3: 表演不靠形容词，靠动作机制

- 明确眼神目标和转移次序。
- 明确手指/手腕/肩肩/重心的具体动作。
- 通过呼吸停顿制造情绪停点。
- 用“开始-掩饰-泄露-收束”替代“高级/高级感”。

Use case:
- 在人物短句里加入“先...后...”和“停住/慢下/回到原位”结构。

## Pattern 4: 表演与场景必须互相影响

- 人物动作由空间压力和道具关系决定。
- 让反射、前景、阴影参与人物关系。
- 让人物不是独立在画面里，而是与入口、出口、距离关系绑定。

Use case:
- 角色关系靠站位与视线而不是对白标签表达。

## Pattern 5: 产品与物件通过机制被理解

- 产品价值通过：擦拭、侧光扫过、遮挡后揭示、镜面/水面反射、按压/碰触造成高光变化。
- 触点固定：手、玻璃、水面、布料、反光面、声音。
- 避免“放在桌上 + 高级”的空泛表述。

Use case:
- 让每个产品动作都有“为什么此刻才露出细节”的机制。

## Pattern 6: 格式一致性与节奏控制

- MV可以高频细节，但保留每个镜头核心。
- Vlog/随手拍要可读的自然偏差：轻微手持偏移、真实反应、非完美取景。
- 每个格式都要保留停顿，没停顿不落地。

Use case:
- 在短时长里也要有可感知起承转合。

## Fail Patterns from weak outputs

- 模板完整但字段空泛。
- 角色表情是标签化词。
- 避免模块缺失。
- 微事件链不成立（起点和落点都没有）。
- 画面变化仅靠“运动”而非关系变化。
